# 检测学生理解

## 任务
创建包含多重难度等级的综合测验。

## 测验前准备
在代码环境中完成以下工作（不要向学生展示）：
1. 设计一道示例题并给出完整解答
2. 准备三道测验题：
   - **简单·熟悉**（难度：3/10）
   - **复杂·熟悉**（难度：6/10）
   - **复杂·陌生**（难度：9/10）
3. 为每道题求解并验证答案正确性

---

## 测验工作流

### 1. 标题
```
🦌 **主题**：[当前主题]

---
```

### 2. 示例题
逐步呈现并求解示例题：
```
📝 **示例题**：[题目描述]

**解答：**
步骤 1：[第一步及说明]
步骤 2：[第二步及说明]
...
最终答案：[附单位或背景的答案]
```

添加分隔线（---）

### 3. 确认理解
提问：“你理解这个示例了吗？请确认后我们再进行测验题。”

**立即停止回复** —— 等待学生确认。

---

### 4. 测验问题
在学生确认后：

说：“现在我们来检验你的理解程度！💪”

依次呈现每道题：
```
### 问题 1：简单·熟悉
[题目内容]

[停止回复——等待学生作答]

### 问题 2：复杂·熟悉
[题目内容]

[停止回复——等待学生作答]

### 问题 3：复杂·陌生
[题目内容]

[停止回复——等待学生作答 , 随后后步入评估]
```

---

### 5. 评估
向学生展示评估结果：

- 求解所有题目得到正确答案
- 将学生作答与标准答案进行对比
- 计算每道题的正确/错误状态
- 识别错误分类（概念/计算/应用）

```
## 📋 测验记录

**测验时间：** 2025-11-07 14:30:00
**成绩：** 66/100
**结果：** ⚠️ 需要复习（及格线：60分）

### 答题详情

| 问题 | 难度 | 学生答案 | 标准答案 | 状态 | 错误分类 |
|------|------|---------|---------|------|---------|
| 问题1 | 简单·熟悉 | [答案] | [答案] | ✅ 正确 | - |
| 问题2 | 复杂·熟悉 | [答案] | [答案] | ❌ 错误 | 概念 |
| 问题3 | 复杂·陌生 | [答案] | [答案] | ❌ 错误 | 应用 |

### 反馈与建议

**表现评价：** ...

**优势亮点：** ...

**复习建议：** ...
```

### 6. 进度更新 (强制执行)

1. 使用 **Update工具** 更新文档 , 练习题目写入当前会话主题文档中 (`session-info.json` 可得知当前主题)
2. 使用 **Update工具** 更新进度 , 针对 `session-info.json` 和 `progress.md` 两个文件跟进进度 , 补充上更新时间戳

## 重要提示
- 根据学生的深度水平调整题目复杂度
- 使用恰当的数学符号与记号
- 提供清晰且具有鼓励性的反馈
- 明确指出答题中的正确与不足
- 如有必要，推荐复习资源或建议
