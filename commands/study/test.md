# 检测学生理解

## 任务
创建包含多重难度等级的综合测验。

## 测验前准备
在代码环境中完成以下工作（不要向学生展示）：
1. 设计一道示例题并给出完整解答
2. 准备三道测验题：
   - **简单·熟悉**（难度：3/10）
   - **复杂·熟悉**（难度：6/10）
   - **复杂·陌生**（难度：9/10）
3. 为每道题求解并验证答案正确性

---

## 测验结构

### 1. 标题
```
🦌 **主题**：[当前主题]

---
```

### 2. 示例题
逐步呈现并求解示例题：
```
📝 **示例题**：[题目描述]

**解答：**
步骤 1：[第一步及说明]
步骤 2：[第二步及说明]
...
最终答案：[附单位或背景的答案]
```

添加分隔线（---）

### 3. 确认理解
提问：“你理解这个示例了吗？请确认后我们再进行测验题。”

**立即停止回复** —— 等待学生确认。

---

### 4. 测验问题
在学生确认后：

说：“现在我们来检验你的理解程度！💪”

依次呈现每道题：
```
### 问题 1：简单·熟悉
[题目内容]

[停止回复——等待学生作答]

### 问题 2：复杂·熟悉
[题目内容]

[停止回复——等待学生作答]

### 问题 3：复杂·陌生
[题目内容]

[停止回复——等待学生作答]
```

---

### 5. 评估
在学生回答全部问题后：

#### 5.1 代码环境工作（后台操作，不向学生展示）

1. **求解与对比**
   - 求解所有题目得到正确答案
   - 将学生作答与标准答案进行对比
   - 计算每道题的正确/错误状态
   - 识别错误分类（概念/计算/应用）

2. **反馈准备**
   - 写出简短反馈评估表现
   - 记录优势与待改进之处
   - 生成总分和及格判定

3. **【新增】数据持久化第一阶段：写入主题文档**

   在当前主题文档（如 `03-函数与模块.md`）末尾添加"📋 测验记录"部分：

   ```markdown
   ## 📋 测验记录

   **测验时间：** 2025-11-07 14:30:00
   **成绩：** 66/100
   **结果：** ⚠️ 需要复习（及格线：60分）

   ### 答题详情

   | 问题 | 难度 | 学生答案 | 标准答案 | 状态 | 错误分类 |
   |------|------|---------|---------|------|---------|
   | 问题1 | 简单·熟悉 | [答案] | [答案] | ✅ 正确 | - |
   | 问题2 | 复杂·熟悉 | [答案] | [答案] | ❌ 错误 | 概念 |
   | 问题3 | 复杂·陌生 | [答案] | [答案] | ❌ 错误 | 应用 |

   ### 反馈与建议

   **表现评价：** ...

   **优势亮点：** ...

   **复习建议：** ...

   **文档状态：** `need_retry`
   ```

   💡 **关键操作：**
   - 若已存在测验记录，则**更新（replace）**而不是追加
   - 确保文档末尾的"文档状态"标记为 `tested` 或 `need_retry`

4. **【新增】数据持久化第二阶段：同步状态文件**

   依次更新（原子操作顺序）：

   ① 更新 `session-info.json` 的 `topicStatus[当前主题]`
   ```json
   {
     "status": "tested",  // 或 "need_retry"
     "testResult": {
       "testTime": "ISO 8601时间戳",
       "score": 66,
       "maxScore": 100,
       "passThreshold": 60,
       "isPassed": false,
       "questions": [...]
     }
   }
   ```

   ② 更新 `progress.md` 的进度表
   ```markdown
   | 03-函数与模块 | need_retry | ... | 66/100 ⚠️ | 需要复习 |
   ```

   ③ 更新时间戳：`timestamps.lastUpdate` = 当前时间

#### 5.2 向学生呈现反馈

向学生展示评估结果（此时已确认数据已保存）：

```
📊 **测验结果与反馈**

**问题 1**：✅ 正确 - [说明]
**问题 2**：❌ 错误 - [说明] | 错误分类：概念理解不足
**问题 3**：❌ 错误 - [说明] | 错误分类：应用能力不足

---

**总体表现**：[总结评估]

**优势亮点**：[学生表现出色之处]

**复习建议**：[需要回顾的主题]

**您的答题记录已自动保存到课程文档中。**

---

💡 **下一步：**
- 使用 **/continue** 复习本主题的关键知识点
- 或使用 **/test** 再次测试以改进成绩
- 对任何题目提出问题以加深理解
```

## 重要提示
- 根据学生的深度水平调整题目复杂度
- 使用恰当的数学符号与记号
- 提供清晰且具有鼓励性的反馈
- 明确指出答题中的正确与不足
- 如有必要，推荐复习资源或建议
