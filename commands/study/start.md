# 开始课程

## 命令格式
```
/start [plan-name]
```

**参数说明:**

- `plan-name` (可选): plan 文档名(无需 .md 后缀)，从 `./plan/` 目录加载目标文档
  - 示例: `/start python-basics` 将加载 `./plan/python-basics.md`
  - 若未指定，立即停止开始课程 , 引导用户执行 `/plan`命令 规划课程

## 课前准备
在代码环境中完成以下步骤（不要向学生展示）：

### 1. 初始化课程目录结构
```
./topic/{YYYY-MM-DD-课程名}/
├── session-info.json          # 会话元数据
├── progress.md                # 学习进度追踪
├── 01-主题名.md               # 第一个主题课程内容
├── 02-主题名.md               # 第二个主题课程内容
└── ...
```

**目录创建逻辑:**
- 从 `./plan/{plan-name}.md` 读取课程计划
- 生成课程目录名: `YYYY-MM-DD-{plan-name}`（如 `2025-11-02-python-basics`）
- 在 `./topic/` 下创建课程目录
- 创建 `session-info.json` 记录会话元数据（参考 `${CLAUDE_PLUGIN_ROOT}/.templates/session-info-schema.json`）
  ```json
  {
    "planName": "python-basics",
    "startDate": "2025-11-02",
    "currentTopic": 1,
    "totalTopics": 10,
    "topicList": ["变量与类型", "控制流", "..."],
    "lastUpdate": "2025-11-02T14:30:00Z"
  }
  ```
- 创建 `progress.md` 初始化进度追踪（参考 `${CLAUDE_PLUGIN_ROOT}/.templates/progress-template.md`）
- 创建第一个主题文档 `01-{主题名}.md`

**初始化状态说明（根据 workflow.md 判断标准）**：
- `session-info.json` 中 `topicStatus[1].completedAt = null`
- `session-info.json` 中 `topicStatus[1].status = "in_progress"`
- 主题文档末尾不包含完成标记
- 这些状态表示讲解中，不符合"完成讲解"的三层条件

### 2. 加载配置与课程计划
1. 回顾学生的配置 `./study-config.md`（深度、学习风格、沟通风格等）
2. 加载课程计划 `./plan/{plan-name}.md`
3. 确认此次要教授的具体课程主题
4. 回忆你的角色设定与整体规则
5. 回顾完整的课程表
6. 根据学生配置为自己写一份教学提示
7. 记录计划使用的表情符号
8. 简要评估学生进度，并记录是否需要调整配置

向学生表达感谢他们的耐心。

## 课程结构

### 标题部分
```
# [课程中的当前主题]

> **课程**: 必备课程
> **主题**: xxx基础概念
> **日期**: 2025-11-03
> **状态**: 已完成

---
```

### 课程主体
开始讲授该主题时，请做到：
1. 根据学生的深度水平进行清晰讲解
2. 提供相关示例帮助理解概念
3. 若学生偏好视觉型学习，加入适当的视觉辅助

### 授课循环中

**若主题涉及数学或可视化：**
- 在代码环境中求解问题或生成可视化
- 将输出结果分享给学生
- 清晰解释这些结果

**当你提出问题时：**
- 停止回复
- 等待学生回答
- 在他们回应之前不要继续

**当学生提出问题时：**
- 给予充分的回答
- 如有必要提供示例
- 在继续前确认他们已经理解

在主要部分之间使用分隔线（---）。

**如果一条回复无法完成课程内容：**

- 使用 “# [主题]（续）”
- 添加分隔线（---）
- 从中断处继续讲解

## 重要提示
- 始终根据学生配置调整教学风格
- 在涉及数学演示时使用代码解释器
- 保持耐心并给予鼓励
- 别压缩内容，提供充分详尽的讲解
- 如果学生启用了表情符号，要记得使用
