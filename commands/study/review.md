# 学习进度回顾

## 命令格式
```
/review [session-dir]
```

**参数说明:**
- `session-dir` (可选): 会话目录名
  - 格式: `YYYY-MM-DD-plan-name`
  - 示例: `2025-11-02-python-basics`
  - 位置: `./topic/` 目录下
  - 若不提供，自动扫描并询问用户选择

## 任务
帮助学生在新会话中快速回顾学习进度，恢复学习上下文，并给出下一步行动建议。

## 工作流程

### 1. 定位会话课程

**如果提供了会话目录：**
1. 查找 `./topic/{session-dir}/` 目录
2. 若不存在，提示用户目录不存在，引导使用 `/plan` 或 `/start`

**如果未提供会话目录（智能模式）：**

1. 扫描 `./topic/` 目录下的所有会话
2. 若无会话存在 → 提示用户尚未开始学习，建议执行 `/plan <主题>`
3. 若只有 1 个会话 → 自动使用该会话
4. 若有多个会话 → 列出所有会话，要求用户选择
   ```
   🦌 检测到以下学习会话：
   
   1.  2025-11-05-python-basics (最近更新: 2025-11-05 14:30)
   2.  2025-11-03-machine-learning (最近更新: 2025-11-03 10:15)
   3.  2025-11-01-web-development (最近更新: 2025-11-01 16:45)
   
   请选择要回顾的课程（输入数字或目录名）：
   ```
   等待用户选择后继续

---

### 2. 读取学习配置

**检查学习配置文件：**

1. 尝试读取 `./study-config.md`
2. 如果文件存在：解析学生的学习偏好 , 其配置将影响后续学习内容的呈现方式
   
3. 如果文件不存在：
   ```
   ⚠️ 检测到您尚未创建学习风格配置。
   
   为了获得最佳的学习体验，请先执行：
   
   /config
   
   这将帮助我了解你的学习偏好（学习深度、学习风格、沟通方式等），
   从而以最适合你的方式呈现回顾内容和教学内容。
   
   配置完成后，请再次使用 /review 命令。
   ```
   停止执行，等待用户完成配置

---

### 3. 读取会话进度信息

**读取会话元数据:**
从 `./topic/{session-dir}/session-info.json` 获取：
```json
{
  "planName": "python-basics",
  "startDate": "2025-11-02",
  "currentTopic": 3,
  "totalTopics": 10,
  "topicList": ["变量与类型", "控制流", "函数", "..."],
  "lastUpdate": "2025-11-05T14:30:00Z",
  "completed": false
}
```

**读取学习进度:**
从 `./topic/{session-dir}/progress.md` 获取：
- 各主题的完成状态
- 完成时间
- 是否进行过测试
- 测试分数或评价

**解析当前状态:**
- 已完成的主题数
- 当前进行中的主题
- 上次学习停止的位置
- 是否有未完成的测验

---

## 重要提示

- 始终根据学生的配置调整语言和语气
- 使用恰当的表情符号以增强可读性（如果学生启用了表情符号）
- 清晰指示下一步的具体命令和说明
- 避免重复已讲解过的内容
- 展现进度成就感，给予鼓励
- 如果学生启用了表情符号，在关键位置使用 🦌（鹿）作为主题图标
