# 规划课程

## 命令格式
```
/plan <topic>
```

**参数说明:**
- `topic` (必需): 学习主题名称
  - 示例: `/plan Python编程基础`
  - 示例: `/plan 微积分入门`
  - 示例: `/plan 机器学习算法`

## 任务
围绕主题：**{{topic}}** 制定一份详尽的课程体系。

## 流程

### 1. 分析阶段
先在代码环境中完成分析（不要向学生展示）：
- 阅读 `./study-config.md` , 回顾学生的配置  （深度、学习风格等）
- 判断一位 {{student_depth}} 水平的学生应学习哪些 {{topic}} 知识
- 找出他们可以进一步探索的高级主题
- 标注课程中需要用到的数学公式

向学生致谢他们的耐心，并告知思考已经完成。

### 2. 制定先修课程
生成通往 {{topic}} 的先修主题：
- 从 **0.1** 开始，一直列到 **0.9**
- 不要包含主课题本身（主课题为 1.0）
- 每一项都应建立必要的基础知识
- 难度需符合 {{student_depth}} 水平

**格式：**
```
# Prerequisite Curriculum

0.1 [主题名称]：[将学习内容的说明]

0.2 [主题名称]：[将学习内容的说明]

...

0.9 [主题名称]：[将学习内容的说明]
```

### 3. 制定主课程
为 {{topic}} 生成主课程：
- 从 **1.1** 起依次列出
- 覆盖该主题的核心内容
- 包含实际应用
- 以复习与评估结束（例如 1.10）

**格式：**

```
# Main Curriculum

1.1 [主题名称]：[将学习内容的说明]

1.2 [主题名称]：[将学习内容的说明]

...

1.10 Review & Assessment：[说明]
```

### 4. 保存并呈现

1. 使用 `Bash` 工具创建目录（如果不存在）：
        ```bash
        mkdir -p plan

2. 必须使用 `Write` 工具将完整课程计划保存到：
      - 文件路径：`./plan/{topic}.md`
      - 文件名规则：保持 topic 原样，仅转为小写
      - 示例: topic="Python编程基础" → `./plan/python编程基础.md`
   - 示例: topic="机器学习算法" → `./plan/机器学习算法.md`

3. 文件内容必须包含：

   ```
   - 完整的必备课程（0.1-0.9）
   - 完整的主要课程（1.1-1.10）
   - 格式化的 Markdown 内容
   ```

4. 向学生展示先修课程与主课程

     - 确认文件保存成功后，告知学生文件位置
     - 展示必备课程和主要课程的概览
     - 以 "请说 `/start {{topic}}` 来开始本次课程计划" 结尾

## 注意事项
- 根据学生配置的深度级别调整内容
- 使用符合其水平的专业术语
- 如学生启用表情符号，记得使用
- 课程要全面，同时保证可消化
